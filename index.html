About CapSavvInator

CapSavvInator is a port of CapSavvy from C#.NET to HTML/JavaScript/PHP.

CapSavvy describes itself as "a tool that can retrieve your data transfer usage from most independent ISPs and display it in your system tray", and CapSavvInator is basically the same thing but it displays your usage on this web page instead of your system tray.

The main purpose of this tool is to provide easy access to your actual and predicted usage data from mobile devices, since they currently don't have anything like CapSavvy available (EDIT: There are now some usage checkers available, but they're not available for all devices, and some actually cost money even though they're based on code found in the free and open-source CapSavvy!)

CapSavvy supports the following ISPs, so in theory I can support all the same ISPs with CapSavvInator, but I only have an API Key for Start (since I'm a Start customer), so that's the only one that is currently supported. If you want one of the others to be supported by this website, you'll need to share your API Key with me (which I'll only use during testing and delete all record of afterwards).

  • Caneris
  • Electronicbox
  • Start
  • TekSavvy
  • Videotron

Privacy & Security

I take the privacy of your Username / API Key very seriously, which is why it is not stored on this web server. When you enter your Username / API Key and click the Save button, your information is saved using Web Storage, which is browser based storage, so your information is stored on your own PC/tablet/phone/whatever and not on this web server. This is so when you come back to the site again, you won't have to re-enter your Username / API Key (at least not on the same device -- if you come back from a different device then you'll need to re-enter it).

After the Username / API Key is saved to your browser's Web Storage area, it is then submitted to this web server by your browser. This submission will use the same protocol that you accessed the site by, so I highly recommend you access the site using HTTPS.

This web server will then securely transmit your Username / API Key to your ISP's web server, which will reply back with your usage information. This web server will then manipulate your usage information (ie calculate average and predicated values) and display it all to you in your browser.
